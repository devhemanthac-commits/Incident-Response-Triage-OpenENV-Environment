✨ Key Features

What makes this environment hard — and useful for evaluating agents.

💥

Cascading Failures

Mis-triaging hard scenarios triggers follow-up alerts — just like production. DB pool exhaustion leads to payment API crash. DNS failure causes total platform outage.

−0.20 penalty per cascade
📈

Time-Series Metrics

Each metric is a 5-point time series (last 5 minutes). Agents must distinguish trends (memory leak: 60% → 78%) from transient spikes.

+0.05 trend analysis bonus
🐺

False Positive Detection

3 "boy who cried wolf" scenarios — nightly backups, staging load tests, canary deployments. High metrics that are expected behavior.

+0.10 bonus for correct low-priority triage
🎯

Expert Team Routing

Each scenario has an optimal team + alternative teams. Routing to an alt team earns partial credit (0.15 vs 0.30). Wrong team = 0.0.

🔄

Multi-Step Episodes

Max 2 attempts per episode. After step 1, agents see consequence observations. Agents that adapt their answer based on feedback earn a bonus.

+0.05 adaptation bonus

📊 Scoring Rubric

Every triage decision is graded across 5 dimensions. Final score capped at [0.0, 1.0].

DimensionMaxRule
Severity0.35Exact match = 0.35 · Off-by-one = 0.15 · Else = 0.0
Team Routing0.30Optimal team = 0.30 · Alt team = 0.15 · Else = 0.0
Escalation0.15Exact match only
Reasoning0.10+0.05 if >30 chars · +0.05 for key indicator mentions
Calibration0.10High confidence when correct · Low confidence when wrong
Trend Analysis Bonus+0.05Reasoning references metric trends (increasing, trending, etc.)
False Positive Bonus+0.10Correct P3/P4 triage on false-positive scenarios
Adaptation Bonus+0.05Improvement from step 1 → step 2 after feedback
Cascade Penalty−0.20Per cascading failure triggered by wrong triage
